# 参与 JoinMesg 贡献

感谢您考虑为 JoinMesg 做出贡献！本文档提供了贡献指南和说明。

## 如何贡献？

### 报告 Bug

在创建 Bug 报告之前，请先检查现有的 Issues 以避免重复。创建 Bug 报告时，请包含：

- 清晰且描述性的标题
- 重现问题的步骤
- 期望的行为与实际的行为
- 服务器版本和插件版本
- 任何相关的错误日志

### 建议新功能

欢迎提出功能建议！请提供：

- 功能的清晰描述
- 该功能的使用场景
- 功能如何工作的示例

### 拉取请求

如果您想贡献代码更改：

1. **Fork 本仓库**
2. **从 main 分支创建新分支**
```bash
git checkout -b feature/你的功能名称
```
3. **按照现有代码风格进行修改**
4. **全面测试您的更改**
5. **提交更改**，使用清晰的提交信息
6. **推送到您的 Fork** 并提交拉取请求

### 代码风格指南

- 遵循项目中现有的代码格式
- 使用有意义的变量和方法名称
- 仅在必要时为复杂逻辑添加注释
- 保持方法的聚焦和简洁

### 开发环境搭建

要搭建开发环境：

1. 克隆仓库
2. 将项目作为 Maven 项目导入
3. 确保已安装 Java 17 或更高版本
4. 使用 Maven 构建项目：
```bash
mvn clean package
```

## 有问题？

如果您对贡献有任何疑问，欢迎创建 Issue！

感谢您帮助改进 JoinMesg！
